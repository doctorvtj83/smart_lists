**Goal:** Closed access via Google login gated by an email allowlist; allowed users are provisioned
just-in-time on first login, while non-allowed users are rejected with a clear message.

**Architecture:** Next.js (App Router, TypeScript) as a single repo for frontend + API. Auth.js
(NextAuth v5) drives the Google OAuth flow; the **allowlist gate** and **JIT user provisioning** live in
isolated, testable functions that the `signIn` callback calls. Persistence via Prisma on Neon Postgres.
Tests run with Vitest against a separate Neon test branch.

**Tech Stack:** Next.js · TypeScript · Auth.js (NextAuth v5) · Google OAuth · Prisma ORM · Neon Postgres · Vitest

These external accounts/keys cannot be created by an agent. The plan assumes them at the relevant points;
set them up beforehand and have the values ready:

1. **Neon account + project** (https://neon.tech): create a Postgres project. You need two connection URLs:

   - **Test branch** (create a branch named `test` in Neon with one click) → for `DATABASE_URL` in `.env.test`.
   - *Why separate:* tests insert rows and truncate tables. That must never touch development/real data.
     A dedicated branch is an isolated copy.
2. **Google OAuth client** (https://console.cloud.google.com → APIs & Services → Credentials →
   "OAuth client ID" → Web application). You need:
   - `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`.
   - Add as an "Authorized redirect URI": `http://localhost:3000/api/auth/callback/google`
     (for local development).

- [ ] **Step 2: Define the data model**

Replace the contents of `prisma/schema.prisma` with:

```prisma
// Prisma schema: describes our DB tables in one central place.
// From this schema Prisma generates the type-safe client (Task 2, Step 5).

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// A signed-in user. Created on first successful login (JIT provisioning).
model User {
  id          String   @id @default(uuid())           // stable UUID (convention: client-generatable)
  googleSub   String   @unique @map("google_sub")      // stable Google identity (OAuth "sub")
  email       String                                   // email from the Google profile (stored normalized)
  displayName String?  @map("display_name")            // display name, optional
  isAdmin     Boolean  @default(false) @map("is_admin") // may maintain the allowlist
  createdAt   DateTime @default(now()) @map("created_at")

  @@map("users")
}

// Email gate for login. Only emails in this table are allowed to sign in.
model AllowlistEntry {
  id        String   @id @default(uuid())
  email     String   @unique                           // always stored normalized (lowercase + trim)
  invitedBy String?  @map("invited_by")                // User.id of the inviter; null for the very first seed
  createdAt DateTime @default(now()) @map("created_at")

  @@map("allowlist_entries")
}
```

> Design notes: `@map`/`@@map` give columns/tables `snake_case` names in the DB while the TypeScript code
> uses `camelCase` — both idiomatic. `email` is stored **normalized**, so the unique index and the login
> comparison are case-insensitive "for free".

```ts
// Checks whether an email is enabled for login.
```

```ts
// Input data for provisioning, derived from the Google profile.
export interface ProvisionInput {
  googleSub: string;
  email: string;
  displayName: string | null;
}

// "Just in time": creates the user on first login, otherwise updates them.
// upsert = update-or-insert in one step, keyed by googleSub (stable identity).
// Idempotent: calling it repeatedly creates no duplicate -> prepares for offline replays (Phase 2).
export async function provisionUser(
  db: PrismaClient,
  input: ProvisionInput,
) {
  const email = normalizeEmail(input.email);
  return db.user.upsert({
    where: { googleSub: input.googleSub },
    update: { email, displayName: input.displayName },
    create: {
      googleSub: input.googleSub,
      email,
      displayName: input.displayName,
    },
  });
}
```

Create `src/auth.ts`:

```ts
import NextAuth from "next-auth";
import Google from "next-auth/providers/google";
import { prisma } from "@/lib/db";
import { isEmailAllowed, provisionUser } from "@/lib/auth/allowlist";

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [Google],
  // JWT session: no session table needed. We put our own user id/isAdmin into the token.
  session: { strategy: "jwt" },
  pages: {
    signIn: "/login", // our own login page instead of the Auth.js default
    error: "/auth/error", // rejected logins land here
  },
  callbacks: {
    // THE ALLOWLIST GATE: false -> login is rejected, redirect to pages.error.
    async signIn({ profile }) {
      const email = profile?.email;
      if (!email) return false;
      if (!(await isEmailAllowed(prisma, email))) return false;

      // Enabled -> create/update the user just in time.
      await provisionUser(prisma, {
        googleSub: String(profile.sub),
        email,
        displayName: (profile.name as string) ?? null,
      });
      return true;
    },

    // On first login, put our DB user id + isAdmin into the JWT (available on every request afterwards).
    async jwt({ token, profile }) {
      if (profile?.sub) {
        const user = await prisma.user.findUnique({
          where: { googleSub: String(profile.sub) },
        });
        if (user) {
          token.userId = user.id;
          token.isAdmin = user.isAdmin;
        }
      }
      return token;
    },

    // Mirror the token values into the session so server/client code can see them.
    async session({ session, token }) {
      if (token.userId) {
        session.user.id = token.userId as string;
        session.user.isAdmin = Boolean(token.isAdmin);
      }
      return session;
    },
  },
});
```

- [ ] **Step 4: Seed script (first admin + their allowlist entry)**

Without an allowlist entry nobody can log in. The seed enables **your** email.

Create `prisma/seed.ts`:

```ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

// Put your own Google email here (normalized, i.e. lowercase):
const ADMIN_EMAIL = "volkertjaden@gmail.com";

async function main() {
  // Create the allowlist entry (idempotent: upsert prevents duplicates on re-seed).
  await prisma.allowlistEntry.upsert({
    where: { email: ADMIN_EMAIL },
    update: {},
    create: { email: ADMIN_EMAIL },
  });
  console.log(`Allowlist enabled: ${ADMIN_EMAIL}`);

  // If the user already exists (after first login), make them admin.
  // On the very first run they don't exist yet -> then nothing happens here, which is fine.
  await prisma.user.updateMany({
    where: { email: ADMIN_EMAIL },
    data: { isAdmin: true },
  });
}

main()
  .then(() => prisma.$disconnect())
  .catch(async (e) => {
    console.error(e);
    await prisma.$disconnect();
    process.exit(1);
  });
```
